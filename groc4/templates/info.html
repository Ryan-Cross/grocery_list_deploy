How did I get here?

My work up until recently was as an industrial electrician. This is a challenging career and it's a good way to earn but I've had my fill of that work.

I'm making the big change to working as a programmer. The logic electricians use for controls and Programmable Logic Controllers is not that different from the coding I've learned and the jump hasn't been bad.

All my studies have been online and self directed. Some of the resources I've used include:

  • I started out with Treehouse.
  • The Python Mastery Course on Udemy
  • Miguel Grinburgs wonderful Flask Course
  • A pile of other places MDN, Youtube, Stack Overflow, Grid By Example, etc etc

Here's the topics I've studied. These are in the order I picked them up. Yeah I was breach.

Python.

I started with learning a language. Why Python? It's easy to pick up, multi-purpose, already one of the major languages and gaining in popularity. I love Python!

Flask web framework.

Django was certainly more popular than Flask. Flask appealed to me because it starts so small but you can grow it as large and complex as you need. I also read that starting with flask would give me a better understanding of backend programming.

Jinja templating and Werkzeug WSGI.

Flask has the great templating of Jinja. Very nice. WSGI is by Werkzeug for development and I switched that for Gunicorn for deployment.

FlaskSQLA: the database and ORM.

FlaskSQLA is the SQLAlchemy package for Flask. I began with some basics of SQL. My first try for an ORM was PeeWee and although it is wonderfully simple there just wasn't enough information available on it. I went next to SQLAlchemy. It's a huge, difficult, exacting package and I payed dearly for the understanding I have. I'm glad I didn't chicken out and go with a no-sql database. I used Postgresql in my database and I am playing with Dbeaver as a gui tool.

Flask wtforms

Flask starts small and you add on as you need functionality. To handle user forms I use wtforms.

Flaskmail.

Flaskmail is the flask package that deals with SMTP and e-mail.

The basics.

Believe it or not I came to these last! I needed to structure the content on the pages I was building and I got drawn in. When I had just started I looked at layout and was repulsed. It was floats etc etc. When I really got into web pages I discovered CSS Grid. What a wonderful bit of gear.

Git and Github for version control.

I'm still learning this. Working alone on small projects doesn't necessitate the use of VCS. I'm doing it as an exercise knowing it will be helpful when I'm a more advanced programmer. I've got the bash basics using the git-bash shell on my windows machine. I try not to go straight to them VCS functionality in the IDE!

IDEs

I began python programming on the shell. Next step was the educational IDE for python "Thonny". Eventually I got Pycharm. Thats where I like to work now. I also really like Atom, that's what I used for learning HTML & CSS.

What next?

I'm going to learn some Javascript next. Photoshop is on the list. Mostly I want to revisit everything I've learned already and really get good at it.
